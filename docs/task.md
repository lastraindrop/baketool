# BakeTool 任务看板

本文档用于记录当前发布窗口内已经完成、仍需执行以及发布后排队的工作。它不是历史档案库，也不是理想化规划表，而是面向当前仓库维护的行动清单。状态说明如下：

- `已完成`：已经落地并与当前代码同步
- `进行中`：当前版本还在收尾或需要持续关注
- `待发布后处理`：不阻断 1.0.0，但应进入后续版本

### 1.2 说明

上面这些事项之所以被定义为“关键阻断项”，是因为它们会直接影响：

- 用户能否顺利使用界面
- 自动化是否可信
- 输出结果是否正确
- 导出流程是否会污染场景状态
- 文档和真实行为是否一致

这些问题如果带着进入正式发布，会在第一轮真实用户使用中迅速暴露，因此已经在当前发布窗口内全部收口。

## 6. 当前判断

从任务视角看，BakeTool 已经完成了“能发布前提下必须完成”的绝大部分工作，剩下的是发布动作本身，而不是新的大规模修复。这一点很重要，因为很多项目会在此阶段不断追加新功能，最终把已经稳定的交付窗口重新拉回不确定状态。

BakeTool 当前更合理的策略是：

1. 保持当前能力面不再继续扩张。
2. 以发布检查清单为准做最终动作。
3. 发布后以 1.0.x 的方式继续稳态修复和增强。

这也是当前看板的设计原则。
